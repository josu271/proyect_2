**PROJECT CLOSEOUT**	

| Project Title:Sistema de Generación Óptima de Horarios Académicos en Entornos de Currículo Flexible (SGDHA) | Date Prepared: 03/07/2026 | Project Manager: Jose Luis Sulla Corveta. |
| :---- | :---- | :---- |

| Project Description El proyecto consistió en el diseño e implementación de una aplicación web inteligente para la generación automática de horarios académicos universitarios, capaz de resolver un problema NP-Hard modelado como un Problema de Satisfacción de Restricciones (CSP). El sistema atiende un problema real de las universidades con currículo flexible, donde la planificación manual de horarios genera solapamientos, incumplimiento de prerrequisitos, excesos en el límite de créditos y baja utilización de aulas y docentes. El entregable clave fue un PMV funcional (v1.0.0) capaz de generar horarios válidos en un tiempo igual o menor a 50 segundos para un escenario de prueba de 200 estudiantes, 10 cursos y 10 docentes, desarrollado bajo metodología Scrum a lo largo de 4 sprints, con stack tecnológico React \+ Vite (frontend), FastAPI (backend), PostgreSQL (base de datos) y OR-Tools / backtracking con forward checking como motor CSP. |

Gestión de Beneficios

| El sistema reduce significativamente el tiempo y esfuerzo manual de planificación de horarios académicos, que actualmente se realiza mediante hojas de cálculo y toma semanas. Con el motor CSP implementado, la generación de un horario completo para un escenario de 200 estudiantes, 10 cursos y 10 docentes toma en promedio 32 segundos. La optimización de carga en el frontend (Sprint 4\) redujo el tiempo de visualización de horarios de 3.2s a 0.8s (-75%), superando el objetivo de ≤1.5s. Estos resultados representan una mejora directa en los KPIs definidos: reducción de conflictos académicos, uso eficiente de aulas y satisfacción estudiantil. |

Necesidades de Negocio

| El proyecto atiende la necesidad institucional de automatizar el proceso de Gestión Académica de las universidades con currículo flexible, evitando: solapamientos de horario (estudiante, docente, aula), incumplimiento de prerrequisitos, excesos en el límite de créditos por semestre (máx. 6\) y desconocimiento de la disponibilidad docente. El sistema entrega, además, control de acceso por roles (administrador, coordinador, docente, estudiante), visualización semanal de horarios, exportación a PDF y un módulo de Green Software para medir el consumo de CO₂ del propio sistema. |
