# Estructura de Testing

Esta carpeta concentra todo el trabajo de pruebas del frontend en una sola ubicacion llamada `test`.

## Subcarpetas principales

- `Aceptacion`: pruebas funcionales hechas con Cypress.
- `E2E`: flujos completos del sistema hechos con Cypress.
- `TDD`: pruebas mas cercanas a logica y rutas renderizadas.
- `Cobertura`: espacio reservado para reportes y seguimiento de cobertura por modulo.
- `support`: helpers compartidos por las pruebas.

## Modulos cubiertos

- `autenticacion`
- `administracion`
- `estudiante`
- `navegacion`
- `multiusuario`
